用于在生产环境进行测试的 Istio 流量镜像功能

Routing rules for HTTP traffic

作者 Christian Posta |  2018年2月8日

  1. 相关内容
该博客文章是在 Istio 0.5 的版本下编写的，因此其中某些内容现在可能已过时。

在非生产/测试环境中，尝试穷举一个服务所有可能的测试用例组合是个令人望而生畏的任务, 在某些情况下，您会发现编写这些用例的所有工作都与实际生产用例不匹配, 理想情况下，我们可以使用实时生产用例和流量来帮助说明我们可能在更人为的测试环境中错过的所测试服务的所有功能区域。

Istio 可以在这里提供帮助, 随着 Istio 0.5.0 的发布，Istio 可以镜像流量来帮助测试您的服务, 您可以编写类似于以下内容的路由规则来启用流量镜像：

apiVersion: config.istio.io/v1alpha2
kind: RouteRule
metadata:
  name: mirror-traffic-to-httbin-v2
spec:
  destination:
    name: httpbin
  precedence: 11
  route:
  - labels:
      version: v1
    weight: 100
  - labels:
      version: v2
    weight: 0
  mirror:
    name: httpbin
    labels:
      version: v2

这里有几点需要注意：

  • 当流量镜像到不同的服务时，会发生在请求的关键路径之外
  • 忽略对任何镜像流量的响应; 流量被视为”即发即忘”
  • 必须创建一个权重为 0 的路由，让 Istio 据此通知 Envoy 创建对应的集群定义; 这应该在未来的版本中解决。

访问镜像任务了解有关镜像的更多信息，并查看更多信息 在我的博客上综合处理这种情况.
