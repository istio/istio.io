    密钥和证书

     阅读大约需要 2 分钟

    如果您怀疑 Istio 使用的某些密钥或证书不正确，那么 第一步是确保 Citadel 健康。

    然后，您可以验证 Citadel 是否实际生成密钥和证书：

    $ kubectl get secret istio.my-sa -n my-ns
NAME                    TYPE                           DATA      AGE
istio.my-sa             istio.io/key-and-cert          3         24d

    其中 my-ns 和 my-sa 是您的 pod 运行的命名空间和 Service Account 。

    如果要检查其他 Service Account 的密钥和证书，可以运行以下命令列出 Citadel为 其生成密钥和证书的所有的 secret：

    $ kubectl get secret --all-namespaces | grep istio.io/key-and-cert
NAMESPACE      NAME                                                 TYPE                                  DATA      AGE
.....

istio-system   istio.istio-cleanup-old-ca-service-account           istio.io/key-and-cert                 3         14d

    确保显示的证书包含有效信息。特别是，Subject Alternative Name 字段应为 URI:spiffe://cluster.local/ns/my-ns/sa/my-sa。 如果不是这种情况，您的 Citadel 可能会出现问题。尝试重新部署 Citadel 并再次检查。

    最后，您可以验证密钥和证书是否由 sidecar 代理正确安装在 /etc/certs 目录中。您可以使用此命令检查：

    $ kubectl exec -it my-pod-id -c istio-proxy -- ls /etc/certs
cert-chain.pem    key.pem    root-cert.pem
